# Configuration

`config.yaml` is generated by `make setup` and gitignored — it stays on your machine. It lives in the project root at `~/ops-nexus/nexus/config.yaml`.

Override the path with the global flag:

```bash
nexus --config /path/to/config.yaml query "..."
```

---

## Full example

```yaml
sources:
  - name: books
    path: ~/Documents/knowledge-drop
    extensions: [.pdf, .md, .txt]
  - name: intelligence
    path: ~/ops-nexus/intelligence
    extensions: [.pdf, .md, .txt]

personal:
  watchDirs:
    - ~/Downloads
    - ~/Desktop
  destDir: ~/Documents/PersonalDocs

postgres:
  dsn: "postgres://vaultuser:${PG_PASSWORD}@localhost:5432/opsnexus?sslmode=disable"

ollama:
  baseURL: http://localhost:11434
  embeddingModel: mxbai-embed-large
  generationModel: llama3.1:8b
  classificationModel: qwen2.5:7b

relevanceThreshold: 0.70
logLevel: info
```

## `sources`

Directories that `nexus ingest` reads from. Each entry has three fields:

| Field | Required | Description |
|---|---|---|
| `name` | yes | Short identifier used in `--source` filters and displayed in results |
| `path` | yes | Absolute or `~`-expanded path to a directory of documents |
| `extensions` | yes | List of file extensions to process (`.pdf`, `.md`, `.txt`) |

**Multiple sources:** you can have as many sources as you want. A common setup:

```yaml
sources:
  - name: books        # technical books, programming references
    path: ~/Documents/knowledge-drop
    extensions: [.pdf, .md, .txt]
  - name: runbooks     # team runbooks and ops docs
    path: ~/ops-nexus/runbooks
    extensions: [.md]
  - name: personal     # this is populated by nexus file / nexus watch
    path: ~/Documents/PersonalDocs
    extensions: [.pdf, .md, .txt]
```

**Notes:**
- Subdirectories are walked recursively
- `nexus ingest` skips files whose SHA-256 hash hasn't changed since last ingest
- The `name` field is stored with each chunk, enabling `nexus query --source books`

## `personal`

Controls the automatic document filing pipeline (`nexus file` and `nexus watch`).

| Field | Required | Description |
|---|---|---|
| `watchDirs` | yes (for watch) | List of directories to watch for new files |
| `destDir` | yes | Root directory where classified documents are moved to |

`destDir` must exist before running `nexus watch`. `make setup` creates `~/Documents/PersonalDocs/` with a standard subdirectory tree.

**`watchDirs`** accepts multiple directories:

```yaml
personal:
  watchDirs:
    - ~/Downloads
    - ~/Desktop
    - ~/Documents/Inbox       # if you have a dedicated drop folder
```

## `postgres`

| Field | Required | Description |
|---|---|---|
| `dsn` | yes | PostgreSQL connection string |

The DSN uses the `${PG_PASSWORD}` placeholder. nexus resolves this from the environment variable at startup. `make setup` writes `export PG_PASSWORD=...` to `~/.zshrc`.

**DSN format:**
```
postgres://user:password@host:port/database?sslmode=disable
```

If you store the password in a secrets manager:

```yaml
postgres:
  dsn: "postgres://vaultuser:${PG_PASSWORD}@localhost:5432/opsnexus?sslmode=disable"
```

Then set `PG_PASSWORD` in your shell. With 1Password CLI:

```bash
export PG_PASSWORD=$(op item get "Local Postgres - opsnexus vaultuser" --fields password --reveal)
```

## `ollama`

| Field | Default | Description |
|---|---|---|
| `baseURL` | `http://localhost:11434` | Ollama server URL |
| `embeddingModel` | `mxbai-embed-large` | Model for embedding text (document chunks and queries) |
| `generationModel` | `llama3.1:8b` | Model for generating answers |
| `classificationModel` | `qwen2.5:7b` | Model for classifying documents in `nexus file` / `nexus watch` |

All three models must be pulled before use. `make setup` does this automatically.

**Changing the generation model:**

You can switch to a different generation model without re-ingesting:

```yaml
ollama:
  generationModel: qwen2.5:14b    # larger model, better answers, slower
```

Or override per-query without changing config:

```bash
nexus query "..." --model qwen2.5:14b
```

**Changing the embedding model:**

> This requires re-ingesting all documents. The embedding dimensions change, making old embeddings incompatible. Run `make setup` which drops and recreates the tables, then `make ingest`.

The default `mxbai-embed-large` is multilingual — Dutch and English documents land in the same vector space, so a query in English finds relevant Dutch content.

## `relevanceThreshold`

| Default | Range | Description |
|---|---|---|
| `0.70` | 0.0–1.0 | Minimum cosine similarity score for a chunk to be included in query results |

Higher = more precise but fewer results. Lower = more results but potentially less relevant.

Per-query override:

```bash
nexus query "..." --threshold 0.55
```

**Guidance by document type:**

| Type | Suggested threshold |
|---|---|
| Dense technical books | 0.70 (default) |
| Short personal documents (invoices, letters) | 0.50–0.60 |
| Mixed knowledge base | 0.65 |
| Exploratory queries where you want breadth | 0.50 |

When a query returns nothing, nexus prints the best score it found:
```
(best match scored 0.62 — threshold is 0.70; try --threshold 0.61 to include it)
```

## `logLevel`

| Default | Options | Description |
|---|---|---|
| `info` | `debug`, `info`, `warn`, `error` | Controls logging verbosity |

| Level | What you see |
|---|---|
| `error` | Only fatal failures |
| `warn` | Non-fatal issues (missing env vars, skipped files) |
| `info` | Per-file milestones: start, done, skipped. Query start/complete. |
| `debug` | Per-stage timing, vector dimensions, chunk counts — the full pipeline trace |

**Override without editing config:**

```bash
NEXUS_LOG_LEVEL=debug nexus query "..."
NEXUS_LOG_LEVEL=debug make ingest
```

**Log format:** coloured terminal output when stdout is a tty; JSON (Loki/Promtail compatible) when piped.

```bash
nexus ingest 2>&1 | grep '"level":"ERROR"'    # parse JSON log in a pipe
```

All log output goes to **stderr**. Answers and table output go to **stdout**. This means you can always redirect cleanly:

```bash
nexus query "..." > answer.txt      # captures only the answer
nexus query "..." 2>/dev/null       # suppresses all logging
```
